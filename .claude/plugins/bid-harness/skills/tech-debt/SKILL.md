Catalog and prioritize the technical debt in this repository.

## Before you begin

Check if `harness-docs/standards-status.md` exists in this repository.
- If it does **not** exist: run `/assess` first to establish a baseline, then return here and continue.
- If it exists: read the rows for `## 03 · Code Quality, Reviews & Standards` to focus the scan on areas already flagged as failing.

## Instructions

1. **Scan the codebase** across three categories:

   **Code smells** — look for:
   - God files: source files over 300 lines containing multiple unrelated responsibilities
   - Long methods: functions over 40 lines
   - Duplication: near-identical logic blocks appearing in different files
   - Magic values: numeric or string literals used directly in logic without named constants

   **Architectural issues** — look for:
   - Circular dependencies between modules (A imports B imports A)
   - Leaking abstractions: direct DB queries or raw HTTP calls scattered through business logic files instead of repository/client layers
   - Inconsistent patterns: the same problem solved in three different ways within the same codebase

   **Dependency health** — inspect the lockfile or manifest (`package.json`, `go.mod`, `pyproject.toml`, `pom.xml`):
   - Major version gaps: packages where a newer major version is widely available
   - Packages historically associated with security issues (note: do not run external scans — flag for the engineer to check)

2. **For each item found**, record:

   | Field | Description |
   |-------|-------------|
   | Category | Code smell / Architectural / Dependency |
   | Location | `file.ts:42` or `module/` |
   | Description | What the problem is, in one sentence |
   | Effort | S < 2h · M half-day · L 1–3 days |
   | Impact | low · medium · high (on maintainability, reliability, or security) |

3. **Write `harness-docs/tech-debt.md`** using the Write tool:

   ```markdown
   # Tech Debt Register

   _Last updated: YYYY-MM-DD. Generated by `/tech-debt`. Edit freely — this is a living document._

   ## Priority Items

   High-impact, low-effort items first:

   | # | Category | Location | Description | Effort | Impact |
   |---|----------|----------|-------------|--------|--------|
   | 1 | Code smell | src/processor.ts:1 | 450-line file with auth, logging, and DB logic mixed | S | High |

   ## Full Catalog

   [All items in the same table format, ordered by impact ÷ effort descending]

   ## Dependencies to Watch

   | Package | Current version | Note |
   |---------|-----------------|------|
   | express | 4.17.1 | v5 released — review migration guide before upgrading |
   ```

4. After writing the file, output a brief summary:

   ```
   Tech debt register written to docs/tech-debt.md.

   Found: X code smells · Y architectural issues · Z dependency notes
   Top priority: [description of #1 item]

   Review docs/tech-debt.md and edit before committing it.
   ```
